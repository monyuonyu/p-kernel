# ステップ3: RTOS タスク管理

p-kernel (micro T-Kernel 2.0) のタスク（スレッド）管理 API を学ぶサンプルです。

## 学べること

- タスクの作成・起動・終了 (`tk_cre_tsk` / `tk_sta_tsk` / `tk_ext_tsk`)
- **FIFO 優先度スケジューリング** — 高優先度タスクが即座に実行される
- **ラウンドロビン (RR) スケジューリング** — 同優先度タスクがタイムスライスで交互実行
- タスク情報の参照と動的変更 (`tk_ref_tsk` / `tk_chg_pri` / `tk_chg_slt`)
- スリープと早期起床 (`tk_slp_tsk` / `tk_wup_tsk`)


## タスクの状態遷移

```
tk_cre_tsk()  tk_sta_tsk()   スケジューラが選択
  ┌──────┐    ┌────────┐    ┌─────────┐
  │DORMANT│───▶│ READY  │───▶│ RUNNING │
  └──────┘    └────────┘    └─────────┘
                   ▲              │
                   │  待ち解除    ▼
                   │          ┌────────┐
                   └──────────│WAITING │  ← tk_slp_tsk() など
                              └────────┘
```

## タスク作成パラメータ (`PK_CRE_TSK`)

| フィールド | 説明 |
|------------|------|
| `task`     | エントリー関数 `void fn(int stacd, void *exinf)` |
| `pri`      | 優先度 1（最高）〜 16（最低）|
| `policy`   | `SCHED_FIFO` または `SCHED_RR` |
| `slice_ms` | RR タイムスライス（ms）。0 = デフォルト 100ms |
| `stksz`    | スタックサイズ（バイト）。0 = デフォルト 4096 |
| `exinf`    | タスクに渡す任意のポインタ |

## スケジューリングポリシーの違い

| ポリシー | 挙動 |
|----------|------|
| `SCHED_FIFO` | 優先度ベース抢占。高優先度タスクが即座に CPU を奪う |
| `SCHED_RR`   | 同優先度の間でタイムスライスを共有して交互実行 |

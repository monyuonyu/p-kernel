# ステップ4: 同期プリミティブ

複数タスクが協調して動くための「同期」の仕組みを学ぶサンプルです。

## 学べること

- **セマフォ** (`tk_cre_sem` / `tk_sig_sem` / `tk_wai_sem` / `tk_del_sem`)
- セマフォの即時ポーリング (`TMO_POL`)
- **イベントフラグ AND 待ち** (`tk_wai_flg` + `TWF_ANDW`)
- **イベントフラグ OR 待ち** (`tk_wai_flg` + `TWF_ORW`)
- フラグのクリア操作 (`tk_clr_flg`)

## セマフォの仕組み

セマフォは「カウント付き信号機」です。

```
初期カウント = 0

タスクA: tk_sig_sem(id, 1)  →  カウント = 1
タスクB: tk_sig_sem(id, 1)  →  カウント = 2
メイン:  tk_wai_sem(id, 2)  →  カウント = 2 以上になったので通過（カウント = 0 に）
```

典型的な使い方: **N 個のタスクの完了を待つ**

| 関数 | 動作 |
|------|------|
| `tk_cre_sem(&csem)` | セマフォ作成。戻り値 = セマフォ ID |
| `tk_sig_sem(id, n)` | カウントを +n する |
| `tk_wai_sem(id, n, tmo)` | カウントが n 以上になるまで待つ |
| `tk_del_sem(id)` | セマフォ削除 |

## イベントフラグの仕組み

イベントフラグは「ビットパターンで複数の条件を同時に管理」できます。

```
初期パターン = 0x00

タスクA: tk_set_flg(id, 0x01)  →  パターン = 0x01
タスクB: tk_set_flg(id, 0x02)  →  パターン = 0x03
メイン (AND待ち): tk_wai_flg(id, 0x03, TWF_ANDW, ...)  →  0x03 が揃ったので通過
```

## 待ちモード

| 定数 | 意味 |
|------|------|
| `TWF_ANDW` | waiptn の**すべてのビット**がセットされるまで待つ |
| `TWF_ORW`  | waiptn の**いずれかのビット**がセットされるまで待つ |

## タイムアウト値

| 値 | 意味 |
|----|------|
| `TMO_POL (0)` | 即時ポーリング（待たない）|
| `TMO_FEVR (-1)` | 永遠に待つ |
| 正の整数 | ミリ秒でタイムアウト |

タイムアウト時の戻り値: `E_TMOUT = -50`
